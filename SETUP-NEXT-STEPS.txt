============================================================
TeachZenith — Setup & Compatibility Check: How to run it
============================================================

WHAT THESE SCRIPTS DO
  setup-teachzenith.sh   Creates the whole "teachzenith" folder tree and all the
                         package.json / tsconfig files.
  check-deps.sh          Installs every dependency and verifies they're compatible
                         (checks Node version, scans for conflicts, type-checks).
  setup-teachzenith.ps1  Windows PowerShell version of the folder setup.

PREREQUISITE (install once)
  - Node.js 18 or newer (LTS recommended): https://nodejs.org
    Check with:  node -v
  Optional, only needed later when you RUN the app (not for this check):
  - PostgreSQL 16+ with pgvector  (to run the database)
  - Redis                          (to run the background worker)

------------------------------------------------------------
ON macOS / LINUX  (and Windows via Git Bash or WSL)
------------------------------------------------------------
1. Put setup-teachzenith.sh and check-deps.sh in the folder where you want the
   project to live (e.g. your Documents or a "projects" folder).

2. Create the structure:
       bash setup-teachzenith.sh

3. Go into the new folder:
       cd teachzenith

4. Install + verify (this is the compatibility check):
       bash check-deps.sh
   You want to see:  "Compatibility check passed. The project is sound."

------------------------------------------------------------
ON WINDOWS (PowerShell, no Git Bash)
------------------------------------------------------------
1. Create the structure:
       powershell -ExecutionPolicy Bypass -File setup-teachzenith.ps1
2. Then run these commands inside the teachzenith folder to install + check:
       cd teachzenith
       npm install
       npm run typecheck
   (Git Bash is smoother — it lets you run check-deps.sh directly. Optional.)

------------------------------------------------------------
AFTER THE CHECK PASSES
------------------------------------------------------------
1. Put the database schema in place: unzip teachzenith-database.zip so that
   db/migrations/, db/seeds/, and db/migrate.sh sit inside the db/ folder.
2. (Optional) drop the harness zip's contents into harness/.
3. When PostgreSQL is installed and running:
       createdb teachzenith
       npm run db:migrate
4. Tell me the check passed, and we move on to backend scaffolding — turning the
   empty backend/ folder into a running API wired to the database.
